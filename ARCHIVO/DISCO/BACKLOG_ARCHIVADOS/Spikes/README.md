## ¿Qué es un Spike?

Un **spike** es una investigación técnica o conceptual con tiempo acotado para:
- Evaluar viabilidad de una propuesta
- Explorar alternativas tecnológicas
- Resolver incertidumbres antes de comprometer recursos

Los spikes que **no se aprueban** se archivan aquí para:
1. Evitar repetir la misma investigación
2. Documentar el razonamiento del descarte
3. Mantener registro histórico de decisiones

## Protocolo de Archivado

1. El spike se crea en `BACKLOG_BORRADORES/` como conversación PO-SM
2. Si se descarta, se mueve a `BACKLOG_ARCHIVADOS/Spikes/`
3. Se añade entrada al INDEX.md de ARCHIVADOS
4. Se genera ficha en INDEX_ABSTRACT.md
